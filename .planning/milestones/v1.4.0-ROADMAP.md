# Milestone v1.4.0: Support GitHub repository

**Status:** ✅ SHIPPED 2026-07-06
**Phases:** 11–13
**Total Plans:** 18

## Overview

Added GitHub repository integration to md2cv — users can now sign in with GitHub via OAuth, pick a repo and branch, auto-pull the open file on load, manually commit changes, see a dirty indicator, and browse the full repo directory tree in a VS Code-style sidebar.

## Phases

### Phase 11: GitHub Auth Foundation

**Goal**: Users can authenticate with GitHub to enable repo access
**Depends on**: Phase 10 (shipped v1.3.0)
**Requirements**: AUTH-01, AUTH-02, AUTH-03
**Plans**: 5 plans

Plans:
- [x] 11-01-PLAN.md — Serverless token-exchange function + env config (.env.example, .gitignore) [wave 1]
- [x] 11-02-PLAN.md — Pure OAuth web-flow lib + unit tests + Vite env typing [wave 1]
- [x] 11-03-PLAN.md — useGitHubAuth hook (callback handling, token persistence, sign-out) [wave 2]
- [x] 11-04-PLAN.md — Header auth UI (button/avatar/dropdown/toast) + App wiring [wave 3]
- [x] 11-05-PLAN.md — Human-verify checkpoint: end-to-end OAuth flow with real credentials [wave 4]

**Details:**
Vercel serverless function `api/github-auth.ts` exchanges OAuth codes for access tokens server-side (client secret never exposed). `VITE_AUTH_ENDPOINT` env var abstracts the token-exchange host so it can move to Cloudflare Worker/Next.js without source changes. `useGitHubAuth` hook handles PKCE-style CSRF state, localStorage token persistence, and sign-out. Header displays 5 states: signed-out, loading, signed-in (avatar/dropdown), error, and dismissible-error.

### Phase 12: Repo Sync & Toolbar

**Goal**: Users can pick a repo and keep the open file in sync with GitHub
**Depends on**: Phase 11
**Requirements**: SYNC-01, SYNC-02, SYNC-03, SYNC-04, SYNC-05, TOOL-01
**Plans**: 7 plans

Plans:
- [x] 12-01-PLAN.md — GitHub repo/content API module + unit tests [wave 1]
- [x] 12-02-PLAN.md — Dialog modal primitive + ConflictModal [wave 1]
- [x] 12-03-PLAN.md — useRepoSync hook (config, auto-pull, dirty, commit, conflict) [wave 2]
- [x] 12-04-PLAN.md — PickerDialog (repo/branch/file) + CommitDialog [wave 3]
- [x] 12-05-PLAN.md — Header File menu + dirty dot + repo caption + toast variants [wave 3]
- [x] 12-06-PLAN.md — App.tsx wiring (hook, dialogs, toasts) [wave 4]
- [x] 12-07-PLAN.md — Human-verify checkpoint: end-to-end sync against live GitHub [wave 5]

**Details:**
`githubRepo.ts` provides 5 async functions: `listRepos`, `listBranches`, `listMdFiles`, `getFileContent`, `commitFile`. `Dialog.tsx` is a reusable modal primitive with full ARIA semantics and keyboard/backdrop dismiss. `useRepoSync` hook manages `RepoConfig` (repo/branch/filePath) + `SyncState` (dirty, conflict, syncing) with localStorage persistence. PickerDialog provides a 3-step repo→branch→file picker; CommitDialog prefills a sensible commit message. Header File menu groups GitHub sync controls (connect, commit, open-picker) alongside import/export .md; PDF export stays in its own group. Dirty dot indicator appears when uncommitted local changes exist.

### Phase 13: File Tree Sidebar

**Goal**: Users can browse the repo directory structure and open any markdown file from a sidebar
**Depends on**: Phase 12
**Requirements**: TREE-01, TREE-02, TREE-03, TREE-04
**Plans**: 6 plans

Plans:
- [x] 13-01-PLAN.md — githubRepo.listTreeEntries (unfiltered tree) + useRepoSync.openFile [wave 1]
- [x] 13-02-PLAN.md — fileTree.ts build/expand logic (TDD) + useRepoTree hook [wave 2]
- [x] 13-03-PLAN.md — FileTree recursive renderer + DirtySwitchDialog [wave 3]
- [x] 13-04-PLAN.md — FileSidebar container (panel/drawer, header, states) [wave 4]
- [x] 13-05-PLAN.md — Header toggle button + App integration (sidebar, tree, dirty-switch, scrim) [wave 5]
- [x] 13-06-PLAN.md — Human-verify checkpoint: end-to-end tree browsing against live GitHub [wave 6]

**Details:**
`listTreeEntries` fetches the full recursive repo tree (all blobs + trees, unfiltered). `buildFileTree` converts GitHub's flat array into nested `TreeNode[]` with folders-first, alphabetical ordering and implied intermediate folders. `pathsToExpand` seeds expand state for the active file's ancestors. `useRepoTree` hook owns fetch lifecycle, expand/collapse state, and active path tracking. `FileTree` renders the tree recursively with VS Code-style chevrons and file icons. `DirtySwitchDialog` guards file switching when the current file has unsaved changes. `FileSidebar` wraps the tree in a resizable panel with loading/empty/error states. Header toggle button persists sidebar open/closed state to localStorage; mobile uses a `scrim` overlay instead of pushing layout.

## Milestone Summary

**Key Decisions:**
- Vercel serverless function for token exchange — avoids exposing `GITHUB_CLIENT_SECRET` in browser; `VITE_AUTH_ENDPOINT` env var satisfies AUTH-02 portability requirement
- `useRepoSync` as the single source of truth for sync state — one hook owns config, dirty flag, conflict, commit; no split across components
- `buildFileTree` sorts folders before files at each level and handles GitHub's truncated-tree flag — resilient to deep repos
- `DirtySwitchDialog` interception pattern — dirty check at navigation time (not commit time) prevents accidental content loss
- File tree opens only `.md` files (TREE-03 scope) — other file types are shown but not clickable

**Issues Resolved:**
- WR-01: Discard button in DirtySwitchDialog was clickable during in-flight commit — disabled during request
- WR-02: Active file ancestor folders were not auto-expanded on tree load — `pathsToExpand` seeded on fetch
- WR-03: `getFileContent` returned empty string instead of throwing on non-200 response — now throws `content_too_large` on 403 Blob responses

**Technical Debt Incurred:**
- `useRepoSync` hook grew to ~300 lines during Phase 12 — consider splitting into `useRepoConfig` (persistence) + `useRepoActions` (sync/commit) in a future refactor
- Tree fetch is a full recursive call on every file open — no diff/cache layer; acceptable for personal tool, but noticeable on repos with thousands of files
- No offline / network-error recovery in file tree — sidebar shows an error state but no retry button
