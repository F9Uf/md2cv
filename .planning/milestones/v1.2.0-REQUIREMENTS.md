# Requirements Archive: md2cv v1.2.0

**Milestone:** v1.2.0 — Support render HTML with Tailwind classes
**Archived:** 2026-04-26
**Status:** ✅ All requirements delivered

---

## Requirements

### Parser

- [x] **PARSER-01**: User's markdown is converted to an HTML string via `md.render()`, replacing the token-walking parser
  - **Outcome:** Validated — parseResume.ts reduced to 5 lines; `md.render()` is the sole parser
- [x] **PARSER-02**: User can write raw HTML elements in markdown and see them rendered in the preview
  - **Outcome:** Validated — `html: true` on MarkdownIt + dangerouslySetInnerHTML in Preview

### Template Styles

- [x] **TMPL-01**: Template styles are defined as a map keyed by HTML element tag (`h1`, `h2`, `h3`, `ul`, `li`, `p`, `a`, `code`, `blockquote`, etc.) with Tailwind class strings per template
  - **Outcome:** Validated — TemplateClasses interface has 13 element keys; CSS also expresses element-level styles via @custom-variant
- [x] **TMPL-02**: All three templates (Classic, Modern, Minimal) are re-expressed in the new element-keyed format
  - **Outcome:** Validated — classic, modern, minimal all in new format; visual output preserved

### Preview Rendering

- [x] **PREV-01**: Preview applies the active template's Tailwind classes to matching HTML elements in the rendered output
  - **Outcome:** Validated — @custom-variant CSS scopes element styles to `.theme-{template}` container
- [x] **PREV-02**: User-authored HTML elements in markdown receive the same element-level Tailwind styling as markdown-generated elements
  - **Outcome:** Validated — same CSS rules apply to all matching tags regardless of origin
- [x] **PREV-03**: User-authored Tailwind utility classes on HTML elements in markdown render correctly at runtime (not limited to build-time scanned classes)
  - **Outcome:** Validated — Play CDN + DOMPurify ADD_ATTR: ['class'] enables arbitrary utility classes
