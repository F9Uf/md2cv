# Requirements Archive: md2cv — Milestone v1.4.0

**Archived:** 2026-07-07
**Milestone:** v1.4.0 Support GitHub repository
**Core Value:** Write your resume in plain Markdown, see it rendered beautifully in real time, export to PDF — zero friction, zero backend.

## v1.4.0 Requirements (All Complete)

### GitHub Authentication

- [x] **AUTH-01**: User can sign in with GitHub via OAuth web flow (token exchange through a Vercel serverless function) — ✓ Delivered Phase 11
- [x] **AUTH-02**: Auth endpoint is abstracted behind config so the token-exchange host can move (Cloudflare Worker, Next.js, etc.) without app-code changes — ✓ Delivered Phase 11 (`VITE_AUTH_ENDPOINT`)
- [x] **AUTH-03**: User can sign out / disconnect GitHub, clearing the stored token — ✓ Delivered Phase 11

### Repo Sync

- [x] **SYNC-01**: User can pick a repository and branch from their own GitHub repos after sign-in — ✓ Delivered Phase 12
- [x] **SYNC-02**: App auto-pulls the latest content of the open file from the selected repo when the app loads — ✓ Delivered Phase 12
- [x] **SYNC-03**: When uncommitted local edits differ from remote at pull time, user is prompted to keep local or take remote — ✓ Delivered Phase 12
- [x] **SYNC-04**: User can commit the current file via a commit button — dialog prompts for a message (sensible default), then commits + pushes via the GitHub API — ✓ Delivered Phase 12
- [x] **SYNC-05**: User can see whether the open file has uncommitted changes (dirty indicator) — ✓ Delivered Phase 12

### File Tree

- [x] **TREE-01**: Left sidebar renders the selected repo's directory tree with nested folders, VS Code-style — ✓ Delivered Phase 13
- [x] **TREE-02**: User can toggle the sidebar (show/hide) — ✓ Delivered Phase 13
- [x] **TREE-03**: Clicking a markdown file in the tree opens it — editor and preview swap to that file's content — ✓ Delivered Phase 13
- [x] **TREE-04**: Only one file is open at a time (opening a file replaces the current one) — ✓ Delivered Phase 13

### Toolbar

- [x] **TOOL-01**: GitHub sync button is grouped/collapsed with the existing import/export .md buttons; PDF export stays a separate button — ✓ Delivered Phase 12

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Multi-file editing (tabs/splits) | Single-file scope confirmed for v1.4.0 — one open file at a time |
| GitHub OAuth Device Flow | Endpoints lack CORS headers; requires proxy with worse UX than web flow |
| PAT-based auth | Superseded by OAuth web flow decision (Vercel function for token exchange) |
| Full backend / Next.js migration | One serverless auth function is sufficient; app stays a static SPA |
| Branch creation / PR workflows | Personal tool — direct commit to the selected branch is enough |
| Non-markdown file editing | md2cv is a markdown resume tool; tree opens .md files only |
