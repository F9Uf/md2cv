# Requirements: md2cv

**Defined:** 2026-04-14
**Core Value:** Write your resume in plain Markdown, see it rendered beautifully in real time, export to PDF — zero friction, zero backend.

## v1 Requirements

### Editor

- [x] **EDIT-01**: User can type markdown in a CodeMirror editor with syntax highlighting — ✓ Phase 2
- [x] **EDIT-02**: Editor updates the preview in real time as user types — ✓ Phase 2

### Preview

- [x] **PREV-01**: Preview renders the markdown as a styled resume layout — ✓ Phase 2
- [x] **PREV-02**: User can switch between Classic, Modern, and Minimal resume templates — ✓ Phase 2
- [x] **PREV-03**: Template switch is reflected instantly in the preview — ✓ Phase 2

### Parsing

- [x] **PARS-01**: h1 parses as candidate name (rendered at top of resume) — ✓ Phase 2
- [x] **PARS-02**: h2 parses as section headers (e.g., Experience, Education) — ✓ Phase 2
- [x] **PARS-03**: h3 parses as entry titles within sections — ✓ Phase 2
- [x] **PARS-04**: Bullet lists under h3 entries parse as detail items — ✓ Phase 2

### Export

- [x] **EXPRT-01**: User can export the current resume preview as a PDF (client-side, print-quality) — ✓ Phase 3
- [x] **EXPRT-02**: User can download the current markdown content as a .md file — ✓ Phase 3

### Storage

- [x] **STOR-01**: Editor content auto-saves to localStorage on change — ✓ Phase 3
- [x] **STOR-02**: Content is restored from localStorage on page load — ✓ Phase 3
- [x] **STOR-03**: User can import a .md file from disk into the editor — ✓ Phase 3

### Layout

- [x] **LAYO-01**: Desktop layout shows editor and preview side-by-side — ✓ Phase 1
- [x] **LAYO-02**: Mobile layout shows tabbed Editor / Preview with toggle — ✓ Phase 1
- [x] **LAYO-03**: A draggable separator divides editor and preview panes; user can drag it to adjust the width ratio — ✓ Phase 1
