# Quick Task 260704-op6: Fix preview/print pagination mismatch under zoom

## Problem

When resume content overflows to multiple pages, the on-screen preview shows wrong page breaks and loses content (whole sections disappear), while browser print output is correct.

## Root cause

Preview and print both use `usePagedjsPreview`, so paged.js input is identical. The difference is the mount:

- `PrintMount` renders into `#print-area` — fixed, 210mm wide, hidden, **unzoomed** (`src/index.css:67-76`). Always correct.
- `Preview` renders into `.pagedjs-scale-wrapper`, which carries CSS `zoom: <scale>` for auto-fit (`src/components/Preview.tsx:100-104`).

The hook deliberately keeps the previous `pageCount` across reflows (to avoid a scale-snap blink), so every re-pagination after the first runs **while the ancestor has `zoom < 1`**. CSS `zoom` affects layout measurement, so paged.js computes break points at scaled coordinates → wrong breaks and dropped nodes on overflow. First render after page load is correct (`pageCount === null` → no zoom yet), matching the observed repro.

## Task 1: Paginate in an unzoomed off-screen staging node, then swap

**File:** `src/hooks/usePagedjsPreview.ts` (only functional change)

- In the render effect, create a staging `<div>` appended to `document.body`, mirroring the `#print-area` cloak: `position:fixed; top:0; left:0; width:210mm; height:auto; visibility:hidden; pointer-events:none; z-index:-1`.
- Call `previewer.preview(wrapper, stylesheets, staging)` instead of rendering into `root`.
- After the flow resolves and `!cancelled`: `root.innerHTML = ''`, `root.append(...staging.children)`, remove staging.
- Remove the `root.innerHTML = ''` at the top of the effect — old pages stay visible until swap (also removes the blink the pageCount-retention hack papered over).
- Cleanup: remove staging alongside the polisher/chunker destroys so cancelled in-flight renders don't leak DOM.
- `PrintMount` and `Preview.tsx` need no changes.

**Verify:** `npx tsc --noEmit` / `npm run build` passes; `npm test` passes; multi-page content paginates identically in preview and print (manual check).

**Done:** Preview page breaks match browser print output when content overflows, including after edits while fit-zoom < 1 is active.
