**Step 2: Create `src/components/Preview.tsx`.**

Per PREV-01: Renders ResumeData as a styled resume layout.
Per D-05: Unrecognized content in `extra` fields rendered as raw HTML via `dangerouslySetInnerHTML`.
Per D-07: Preview scrolls independently (the parent SplitPane already has `overflow-auto` on the right pane).

**Component interface:**

```typescript
interface PreviewProps {
  resumeData: ResumeData
  template: TemplateName
}

export default function Preview({ resumeData, template }: PreviewProps)
```

**Rendering logic:**

1. Get styles: `const styles = TEMPLATE_STYLES[template]`
2. If `resumeData.name` is empty string and `resumeData.sections.length === 0`, show an empty state: a centered gray message "Start typing markdown to see your resume preview" in a `div` with `className="flex items-center justify-center h-full text-gray-400 text-sm"`.
3. Otherwise render:
   ```
   <div className={styles.container}>
     {resumeData.name && <h1 className={styles.name}>{resumeData.name}</h1>}
     {resumeData.preamble && <div className={styles.preamble} dangerouslySetInnerHTML={{ __html: resumeData.preamble }} />}
     {resumeData.sections.map((section, si) => (
       <div key={si}>
         <h2 className={styles.sectionHeading}>{section.heading}</h2>
         {section.extra && <div className={styles.extra} dangerouslySetInnerHTML={{ __html: section.extra }} />}
         {section.entries.map((entry, ei) => (
           <div key={ei}>
             <h3 className={styles.entryTitle}>{entry.title}</h3>
             {entry.extra && <div className={styles.extra} dangerouslySetInnerHTML={{ __html: entry.extra }} />}
             {entry.details.length > 0 && (
               <ul className={styles.detailList}>
                 {entry.details.map((detail, di) => (
                   <li key={di} className={styles.entryDetail}>{detail}</li>
                 ))}
               </ul>
             )}
           </div>
         ))}
       </div>
     ))}
   </div>
   ```

IMPORTANT: The `dangerouslySetInnerHTML` usage is safe here because the HTML comes from markdown-it with `html: false` (default), which escapes all user HTML. The parser (Plan 01) must NOT enable `html: true`. See threat model T-02-04.

State management:
```typescript
// Template state with localStorage persistence (per D-09)
const [selectedTemplate, setSelectedTemplate] = useState<TemplateName>(() => {
  try {
    const stored = localStorage.getItem('md2cv-template')
    if (stored === 'classic' || stored === 'modern' || stored === 'minimal') return stored
  } catch { /* ignore */ }
  return 'classic'
})

// Markdown content state (initialized with sample resume per D-02)
const [markdownContent, setMarkdownContent] = useState<string>(SAMPLE_RESUME)

// Debounced resume data for preview (per D-06)
const [resumeData, setResumeData] = useState<ResumeData>(() => parseResume(SAMPLE_RESUME))

// Debounce timer ref
const debounceRef = useRef<ReturnType<typeof setTimeout>>()

// Handle markdown changes with ~150ms debounce
const handleMarkdownChange = useCallback((value: string) => {
  setMarkdownContent(value)
  if (debounceRef.current) clearTimeout(debounceRef.current)
  debounceRef.current = setTimeout(() => {
    setResumeData(parseResume(value))
  }, 150)
}, [])

// Persist template selection (per D-09)
const handleTemplateChange = useCallback((template: TemplateName) => {
  setSelectedTemplate(template)
  try { localStorage.setItem('md2cv-template', template) } catch { /* ignore */ }
}, [])

// Cleanup debounce on unmount
useEffect(() => {
  return () => { if (debounceRef.current) clearTimeout(debounceRef.current) }
}, [])
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User markdown -> dangerouslySetInnerHTML | Parser-rendered HTML injected into DOM via React's dangerouslySetInnerHTML |
| localStorage -> template state | Template name read from localStorage used to select style classes |
| localStorage -> template write | Selected template name written to localStorage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Injection (XSS) | Preview.tsx dangerouslySetInnerHTML | mitigate | HTML comes from markdown-it with `html: false` (default), which escapes `<script>`, `<img onerror>`, etc. Plan 01 parser MUST NOT enable `html: true`. The `extra` fields only contain markdown-it's safe rendered output (escaped entities). Verify in code review that `new MarkdownIt()` is called without html option. |
| T-02-07 | Tampering (T) | localStorage md2cv-template | mitigate | Validate stored value against allowlist: only accept 'classic', 'modern', 'minimal'. Any other value falls back to 'classic'. Implemented in App.tsx useState initializer with explicit `=== 'classic' || === 'modern' || === 'minimal'` check. |
| T-02-08 | Denial of Service (D) | Debounce / large markdown | accept | A user could paste extremely large markdown causing parser lag. Acceptable risk for personal tool -- user controls their own input. 150ms debounce prevents rapid-fire parsing. |
| T-02-09 | Information Disclosure (I) | localStorage template key | accept | Template preference stored in localStorage is not sensitive data. |
</threat_model>
