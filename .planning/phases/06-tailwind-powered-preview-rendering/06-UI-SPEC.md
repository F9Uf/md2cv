# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Scope Note

This phase has **no new UI surface**. It is a styling architecture migration: element-level styles move from TypeScript class strings in `templateStyles.ts` to scoped CSS files using `@apply`, and the Tailwind Play CDN is added to `index.html` so user-authored utility classes resolve at runtime. The resume preview appearance stays identical to Phase 5 output. The editor, header, tabs, and split-pane layout are untouched.

All design contract values below document the **existing preview rendering contract** that must be preserved after migration, not new choices.

## CSS Architecture Contract

This section is specific to Phase 6 and consumed by the executor as the primary technical contract.

### Theme CSS Output

Each theme's element styles are expressed as scoped `@apply` rules wrapped in `@layer base`. The theme class on the container div scopes all selectors.

Container class naming convention: `theme-{templateName}` (e.g., `theme-classic`, `theme-modern`, `theme-minimal`).

Example structure (Classic):

```css
@layer base {
  .theme-classic h1 { @apply text-xl font-bold text-center mb-1; }
  .theme-classic h2 { @apply text-base font-bold uppercase border-b border-black pb-1 mt-4 mb-3; }
  .theme-classic h3 { @apply text-sm font-semibold mt-3; }
  .theme-classic p  { @apply text-center text-xs text-gray-600 mb-4; }
  .theme-classic ul { @apply list-disc pl-2 mt-1 space-y-0.5; }
  .theme-classic ol { @apply list-decimal pl-2 mt-1 space-y-0.5; }
  .theme-classic li { @apply text-xs ml-4; }
  .theme-classic code { @apply font-mono text-xs bg-gray-100 px-0.5 rounded; }
  .theme-classic pre  { @apply font-mono text-xs bg-gray-100 p-3 rounded mt-2 overflow-x-auto; }
  .theme-classic a    { @apply text-blue-700 underline; }
  .theme-classic blockquote { @apply border-l-2 border-gray-400 pl-3 text-xs text-gray-600 italic my-2; }
  .theme-classic hr { @apply border-t border-gray-300 my-4; }
}
```

Modern and Minimal follow the same pattern using their respective class strings from `templateStyles.ts`.

### Preview.tsx Container

After migration, `Preview.tsx` applies:

```tsx
<div className={`theme-${template} ${styles.container}`} dangerouslySetInnerHTML={{ __html: htmlContent }} />
```

Where `styles.container` is now only the layout string (e.g., `font-serif max-w-[800px] mx-auto p-8 leading-snug`).

### Tailwind Play CDN

Added to `index.html` `<head>` without `defer` conflicts with the build-time Vite processing:

```html
<script src="https://cdn.tailwindcss.com"></script>
```

Placement: before `</head>`, after existing meta tags. The Play CDN resolves user-authored utility classes (e.g., `<span class="text-red-500">`) in the preview's `dangerouslySetInnerHTML` content at runtime.

### Cascade Priority (enforced structurally)

```
@layer base    ← theme element styles (.theme-classic h1 { ... })
@layer utilities ← Tailwind utilities (Play CDN output)
```

User-authored classes always win. No `!important` needed.

## Interaction Contract

No new interactions. The template switcher (Classic / Modern / Minimal) already exists in `Header.tsx`. After migration, switching templates:

1. Updates the `template` prop passed to `Preview.tsx`
2. `Preview.tsx` applies `theme-{template}` class to the container div
3. CSS cascade applies the correct scoped element styles
4. No JavaScript class injection, no DOM walking

The interaction is identical to Phase 5 — only the styling mechanism changes.
