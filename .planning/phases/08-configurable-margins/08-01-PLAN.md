<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → App state | Persisted margin values are read back on load; malformed JSON or out-of-range values must not corrupt state |
| User input → margin state | onChange fires on every keystroke; NaN and out-of-range values must be rejected before updating state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | localStorage `md2cv-margins` | mitigate | Validate each value on load: must be number in [0,50]; fall back to 15 per side if invalid |
| T-08-02 | Tampering | MarginControls onChange | mitigate | Guard: `if (!e.target.value \|\| isNaN(parsed)) return`; clamp `Math.min(50, Math.max(0, parsed))` before setState |
| T-08-03 | Denial of Service | localStorage write on every keystroke | accept | Existing md2cv-content write also fires on every debounce tick; same risk profile, no new attack surface |
</threat_model>
