  <action>
    Create `api/github-auth.ts` as a self-contained Vercel Node serverless function with NO imports (uses global `fetch`, Node 20). Implement EXACTLY this behavior:

    ```typescript
    // Minimal structural types — api/ is not in tsc build scope and @vercel/node is not installed.
    interface AuthRequest {
      method?: string
      body: { code?: string } | string | undefined
    }
    interface AuthResponse {
      status: (code: number) => AuthResponse
      setHeader: (name: string, value: string) => void
      json: (body: unknown) => void
      end: () => void
    }

    export default async function handler(req: AuthRequest, res: AuthResponse) {
      res.setHeader('Access-Control-Allow-Origin', '*')
      res.setHeader('Access-Control-Allow-Methods', 'POST, OPTIONS')
      res.setHeader('Access-Control-Allow-Headers', 'Content-Type')

      if (req.method === 'OPTIONS') { res.status(204).end(); return }
      if (req.method !== 'POST') { res.status(405).json({ error: 'method_not_allowed' }); return }

      const body = typeof req.body === 'string' ? JSON.parse(req.body) : req.body
      const code = body?.code
      if (!code) { res.status(400).json({ error: 'missing_code' }); return }

      const clientId = process.env.GITHUB_CLIENT_ID
      const clientSecret = process.env.GITHUB_CLIENT_SECRET
      if (!clientId || !clientSecret) { res.status(500).json({ error: 'server_misconfigured' }); return }

      let data: { access_token?: string; error?: string }
      try {
        const ghRes = await fetch('https://github.com/login/oauth/access_token', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json', Accept: 'application/json' },
          body: JSON.stringify({ client_id: clientId, client_secret: clientSecret, code }),
        })
        data = await ghRes.json()
      } catch {
        res.status(502).json({ error: 'exchange_failed' }); return
      }

      if (data.error || !data.access_token) {
        res.status(502).json({ error: data.error || 'exchange_failed' }); return
      }
      // Return ONLY the user's access token. Never echo client_secret or the raw GitHub response.
      res.status(200).json({ access_token: data.access_token })
    }
    ```

    Do not add logging of the request body or secrets. The `repo` scope (D-11) is requested client-side in the authorize URL, not here.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> serverless function | Untrusted `code` arrives from the client; the function holds the confidential client secret |
| serverless function -> github.com | Server-to-server exchange over HTTPS carrying the client secret |
| repo -> git remote | Env files could accidentally carry real secrets into version control |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | `GITHUB_CLIENT_SECRET` | mitigate | Secret read only from `process.env` server-side; never VITE-prefixed; never in a response body or log (Task 2). Enforced by verify: no `VITE_*SECRET` in `.env.example`. |
| T-11-02 | Information Disclosure | `.env` files | mitigate | `.gitignore` ignores `.env`/`.env.*` while keeping `.env.example` (Task 1). |
| T-11-03 | Tampering / Injection | request body `code` | mitigate | `code` validated present (400 if missing) and passed as a JSON field, not string-concatenated into the GitHub URL (Task 2). |
| T-11-04 | Denial of Service | open CORS `*` | accept | Personal tool, single user; wildcard CORS acceptable. Endpoint only performs an exchange that requires a valid GitHub-issued `code` bound to the registered client id. |
| T-11-05 | Elevation of Privilege | `repo` scope breadth | accept | `repo` scope required for Phase 12 private-repo commits (D-11); documented and intentional. |
</threat_model>
