| rendered header text -> DOM | Username string from GitHub rendered as text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-13 | Tampering / XSS | username + avatar render | mitigate | Username rendered as React text child (auto-escaped); avatar set via `src` attribute only (no `dangerouslySetInnerHTML`). Img `onError` fallback avoids broken-image info leak. |
| T-11-14 | Information Disclosure | token/error UI | accept | No token value is ever rendered; only `login`/`avatar_url` and generic error copy shown. Error strings are static (no server internals). |
| T-11-15 | Spoofing | sign-out affordance | mitigate | Sign-out is an explicit `role="menuitem"` action calling `onSignOut` (clears token in 11-03); no cross-origin trigger path. |
</threat_model>
