## Scope of New UI

Phase 11 adds three elements — all within the existing `h-12 bg-gray-900` header. Nothing outside the header changes.

| Element | Condition |
|---------|-----------|
| "Sign in with GitHub" button | signed-out state |
| Loading state on the button | token exchange in progress |
| Circular avatar | signed-in state |
| Avatar dropdown (username + Sign out) | signed-in, on avatar click |
| Inline error toast (floating) | auth failure |

---

## Visual Hierarchy

Signed-out primary anchor: the "Sign in with GitHub" button at the right of the header controls — it is the only interactive auth element visible and draws the eye via contrast against `bg-gray-900`.

Signed-in primary anchor: the circular avatar button at the same rightmost position — it replaces the sign-in button one-for-one and is the sole entry point to auth actions.

The "Export PDF" blue button retains its existing primary status on the action spectrum; all auth UI uses `bg-gray-700` to avoid competing with it.

## Spacing Scale

Declared values (all multiples of 4). Existing header usage listed where relevant.

| Token | px value | Tailwind class | Usage in Phase 11 |
|-------|----------|---------------|-------------------|
| xs | 4px | gap-1 | Icon-to-text gap inside sign-in button |
| sm | 8px | gap-2 | Spacing between header controls (already in use) |
| md | 16px | px-4 | Header horizontal padding (already in use) |
| button-h | 32px | h-8 | Sign-in button height; avatar height (matches existing buttons) |
| header-h | 48px | h-12 | Header height (existing, unchanged) |

Exceptions:
- Avatar: 32px × 32px (h-8 w-8 rounded-full) — fits existing button height, no exception needed
- Dropdown min-width: 160px (min-w-[160px]) — accommodates typical GitHub usernames
- Dropdown item vertical padding: 8px top + 8px bottom (py-2) per item
- **12px horizontal padding (px-3):** Used on sign-in button, dropdown items, and error toast. Matches the existing gray button pattern in `Header.tsx` — deviating would break visual consistency with the existing button row. Justified exception; not added to the scale table.

## Typography

Phase 11 introduces only two weights: regular (400) for button/body text and semibold (600) for the dropdown username label. All new elements are at the button/label scale.

| Role | Size | Tailwind | Weight | Weight class | Line Height |
|------|------|----------|--------|--------------|-------------|
| Button / body text | 14px | text-sm | 400 — regular | font-normal | 1.5 |
| Dropdown username label | 14px | text-sm | 600 — semibold | font-semibold | 1.5 |

> The existing "md2cv" branding text in the header uses `text-lg font-bold` (18px / 700). That element is unchanged by Phase 11 and is therefore not part of this phase's typography contract. It is noted here only for completeness — it does not affect Phase 11 implementation.

## Color

All new Phase 11 UI is within the dark `bg-gray-900` header. Color contract is scoped to the header surface.

| Role | Tailwind class | Usage |
|------|---------------|-------|
| Dominant (60%) | bg-gray-900 | Header surface; dropdown panel background |
| Secondary (30%) | bg-gray-700 | Sign-in button face; dropdown item hover state |
| Accent (10%) | bg-blue-600 / ring-blue-400 | PDF Export button only; avatar focus ring only |
| Error | text-red-400 | Error toast text |
| Error surface | bg-red-950 / border-red-800 | Error toast container |

**Accent reserved for (explicit list):**
- The existing "Export PDF" button — `bg-blue-600`
- Avatar hover/focus ring — `ring-blue-400` (subtle affordance that the avatar is clickable)

Accent is NOT used for: the sign-in button, dropdown items, or error states. All auth UI uses the secondary gray-700 styling to avoid competing with the primary blue PDF export action.

**Border colors:**
- Header elements: `border-gray-600`
- Dropdown panel border: `border-gray-600`
- Dropdown internal divider (below username): `border-gray-700`

**Text colors within header/dropdown:**
- Primary text: `text-white`
- Secondary / muted text (e.g., error): `text-gray-300`
- Error text: `text-red-400`

## Component Interaction States

### State 1: Signed-out

Button position: rightmost in the header controls `div` (after "Export PDF").

```
h-8 px-3 rounded bg-gray-700 text-white text-sm border border-gray-600
hover:bg-gray-600 transition-colors flex items-center gap-1
```

- GitHub mark SVG: 16px × 16px, `fill-white`, inline SVG, no external dependency
- Icon left of label text: `<svg …/> Sign in with GitHub`
- aria-label: "Sign in with GitHub"

### State 2: Loading (token exchange in progress)

Same button dimensions; disabled while exchange is in flight.

```
h-8 px-3 rounded bg-gray-700 text-white text-sm border border-gray-600
opacity-75 cursor-not-allowed flex items-center gap-1
```

- GitHub mark SVG replaced with: 16px CSS `animate-spin` spinner (border-2 border-white border-t-transparent rounded-full w-4 h-4)
- Button label text: "Signing in..."
- aria-label: "Signing in, please wait"
- aria-busy: true
- Implemented by: disabled attribute + opacity-75

### State 3: Authenticated — avatar button

Replaces the "Sign in with GitHub" button in the same position.

```
h-8 w-8 rounded-full overflow-hidden cursor-pointer
ring-1 ring-gray-600 hover:ring-2 hover:ring-blue-400
hover:ring-offset-1 hover:ring-offset-gray-900 transition-all
focus:outline-none focus:ring-2 focus:ring-blue-400
```

- `<img>` src from GitHub `/user` response `avatar_url`
- Fallback if image fails: `bg-gray-600 flex items-center justify-center text-white text-xs font-semibold` showing first character of login
- aria-label: "GitHub account menu"
- aria-haspopup: "menu"
- aria-expanded: true/false

### State 4: Dropdown menu (signed-in, avatar clicked)

Positioned absolute below the avatar, right-aligned.

```
absolute right-0 top-full mt-1 min-w-[160px] z-50
bg-gray-900 border border-gray-600 rounded shadow-lg
```

Structure (top to bottom):

**Username row (non-interactive):**
```
px-3 py-2 text-sm font-semibold text-white
border-b border-gray-700
```
Content: `{username}` from GitHub API (no @ prefix)

**"Sign out" item (interactive):**
```
px-3 py-2 text-sm text-gray-200
hover:bg-gray-700 cursor-pointer rounded-b
```
Content: "Sign out"
role: "menuitem"
aria-label: "Sign out of GitHub"

Dropdown closes on: Sign out click, click outside, Escape key.

### State 5: Auth failure toast

Floats below the header, right-aligned (does not expand the h-12 header).

Position: `fixed top-12 right-4 z-50` (appears immediately below the header, right edge)

```
flex items-center gap-2 px-3 py-2 rounded
bg-red-950 border border-red-800 text-red-400 text-sm
shadow-md max-w-[320px]
```

- Error message text (see Copywriting Contract)
- Dismiss button: `×` character, `ml-auto pl-2 text-red-400 hover:text-white cursor-pointer`
- aria-live: "polite" on the container so screen readers announce it
- Auto-dismiss: none (dismissible only — user clicks ×)
- Retry: user clicks "Sign in with GitHub" button (which is still visible behind the toast)

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA | Sign in with GitHub | CONTEXT.md D-09 |
| Loading button label | Signing in... | Claude's discretion |
| Dropdown username | {username} (e.g. charnsak) | GitHub GET /user .login field |
| Dropdown sign-out item | Sign out | CONTEXT.md D-08 |
| Auth failure — user denied OAuth | GitHub access was denied. Click Sign in to try again. | Claude's discretion |
| Auth failure — state mismatch (CSRF) | Sign-in failed. Please try again. | Claude's discretion |
| Auth failure — server/exchange error | Sign-in failed. Please try again. | Claude's discretion |
| 401 on token validation | (silent) — clear token, return to signed-out state | CONTEXT.md D-06 |

**Destructive actions in Phase 11:**

| Action | Element | Confirmation approach |
|--------|---------|----------------------|
| Sign out | "Sign out" dropdown item | Single click — no dialog. Action is reversible (click Sign in again). | 

> No destructive confirmation dialog. Sign-out in a personal tool does not warrant a modal. The dropdown item is explicit enough.

**localStorage key:** `md2cv-github-token` — follows existing `md2cv-*` prefix convention. (CONTEXT.md — Claude's discretion on key naming.)

## Accessibility Notes

- All interactive header elements have `aria-label`
- Dropdown follows ARIA `role="menu"` + `role="menuitem"` pattern
- Error toast uses `aria-live="polite"` for screen reader announcement
- Focus is returned to the avatar button when dropdown closes via Escape
- Avatar fallback character is announced via `aria-label` on the fallback element
- Spinner state sets `aria-busy="true"` on the button
