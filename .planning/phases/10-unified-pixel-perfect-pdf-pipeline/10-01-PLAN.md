<objective>
Make the exported PDF originate from the same paged.js DOM the user sees on screen by:
1. Flipping the always-mounted `#print-area` `<Preview>` in `App.tsx` from `enablePagination={false}` to `enablePagination={true}` so the print-mount runs its own paged.js Previewer (D-01, D-02).
2. Off-screen-positioning `#print-area` on screen media so paged.js has real layout to measure, and neutralizing that offset inside `@media print` so the print engine sees normal flow (D-03).
3. Deleting the static `@page { margin: 15mm }` in `src/index.css` and replacing it with `@page { size: A4 portrait; margin: 0 }` so the browser print engine does not add a second gutter on top of paged.js's per-page margins (D-04, D-04a).
4. Verifying — without modification — that the Phase 9 `.pagedjs-scale-wrapper` defensive print reset in `pages.css` is preserved (D-06: load-bearing for mobile PDF parity), and that the `.pagedjs_page { margin-bottom: 0 }` line already required by D-07 is intact.

Purpose: PDFX-01 (pixel-perfect parity preview ↔ PDF) and PDFX-02 (single unified rendering path) — the user's exported PDF is literally a rasterization of the on-screen paginated DOM, not a parallel render.

Output: A working `npm run build` + manual export produces a PDF whose page count, page-break positions, margins, fonts, colors and per-template Tailwind classes match the on-screen preview at 100% across all three templates.
</objective>

Write this exact content to `src/index.css`:

```css
@import "tailwindcss";

@media print {
  /* Hide the entire app shell (header, editor, splitter) */
  #app-shell {
    display: none !important;
  }

  /* Hide the hidden file input */
  input[type="file"] {
    display: none !important;
  }

  /* Show the print-area in normal flow (neutralize the off-screen offset
   * used on screen so paged.js has real layout to chunk against — see the
   * screen-side rule below). Per D-03. */
  #print-area {
    display: block !important;
    position: static;
    left: auto;
    width: 100%;
    margin: 0;
    padding: 0;
  }

  /* Tell the browser print engine the physical page is A4 with NO gutter.
   * Paged.js already baked the user's per-side margins inside each
   * .pagedjs_page; a non-zero gutter here would double them. Per D-04a. */
  @page {
    size: A4 portrait;
    margin: 0;
  }

  body {
    background: white !important;
    margin: 0;
    padding: 0;
  }
}

/* Park the print-area off-screen on screen media. Paged.js needs the element
 * to have real layout (not display:none) so it can measure .pagedjs_page
 * dimensions correctly. Per D-03. */
#print-area {
  position: absolute;
  left: -9999px;
  top: 0;
  width: 210mm;
}
```
