  <how-to-verify>
**Setup verified by Task 1** — multi-page resume loaded, dev server on `http://localhost:5173`.

Run the following matrix. For each cell, capture pass/fail and any visual diff notes.

### Test matrix

For each template (Classic, Modern, Minimal):

1. **Switch template** via the header template selector.
2. **Confirm on-screen pill** reads "Page X of N" with N ≥ 2. Note the value of N.
3. **Open the browser print dialog** by clicking "Export PDF" in the header.
   - In the print dialog → "Save as PDF" → set scale to 100% (do NOT use "Fit to printable area").
   - Confirm the **printed page count in the dialog preview equals N** (the on-screen count). If they differ → FAIL, log the deviation.
4. **Visual diff page 1** preview vs print preview:
   - Same name, same section headings, same body text, same colors, same fonts.
   - Margins (top/right/bottom/left) match what MarginControls currently shows.
   - No "Page X of N" pill in the PDF preview.
   - No drop shadow / border under the page in the PDF preview.
   - No gray surround in the PDF preview.
5. **Visual diff page 2** (and any subsequent pages):
   - The break in the PDF happens at the **same paragraph/entry boundary** as on-screen at 100% zoom.
   - Header repetition, bullet continuation, and visual rhythm are identical.
6. **Save the PDF** to a tmp file (suggested name: `verify-{template}-{date}.pdf`). Open it in the OS PDF viewer (Preview.app / Acrobat) and spot-check: text is selectable, fonts render natively (not raster), colors match, no clipping at page edges.

### Margin propagation check (one template only — pick Classic)

7. In MarginControls, change `top` from 15 to 30 (or any large delta).
8. Wait ~200ms for paged.js to reflow both Previewers.
9. Export PDF again. The PDF's top gutter must visibly grow to match the new on-screen top margin. If the PDF top gutter is unchanged or doubled (30 + 15 hardcoded) → FAIL — the `@page { margin: 15mm }` rule was not fully removed.
10. Reset top margin to 15.

### Mobile A4 capture check

11. Open DevTools → Device Toolbar → select an iPhone or Android mobile preset (≤ 767px width).
12. Confirm the on-screen preview rendered at `zoom: 0.5` (it should look ~half-size relative to the viewport).
13. Click Export PDF. The print dialog preview must show A4 pages at **full size** (210mm × 297mm), not half-size. If the PDF preview is half-size or otherwise scaled → FAIL — the Phase 9 `.pagedjs-scale-wrapper { zoom: 1 !important }` rule is not winning the cascade.
14. Save the mobile-export PDF and confirm in the OS PDF viewer that pages measure A4.

### No-chrome check

15. In the final PDF, confirm there is NO:
    - "Page X of N" pill (in any corner of any page).
    - Drop shadow around any page.
    - Gray background behind any page.
    - Visible blank/duplicate page from the on-screen mount bleeding through.

### Doc-line spot check

16. Open README.md and confirm line 32 reads `**paged.js + browser print** — paginated DOM rendered in-browser, exported via the browser's native Save-as-PDF`.
17. Open CLAUDE.md and confirm line 14 reads `- **Tech stack**: markdown-it for parsing, CodeMirror for editor, paged.js for pagination, browser print for PDF export`.
