```typescript
function loadRepoConfig(): RepoConfig | null {
  try {
    const raw = localStorage.getItem(REPO_CONFIG_KEY)
    if (!raw) return null
    const p = JSON.parse(raw)
    if (typeof p.owner === 'string' && typeof p.repo === 'string' && typeof p.branch === 'string'
      && typeof p.defaultBranch === 'string' && typeof p.filePath === 'string') return p as RepoConfig
  } catch { /* ignore */ }
  return null
}
function loadSyncState(): SyncState | null {
  try {
    const raw = localStorage.getItem(SYNC_STATE_KEY)
    if (!raw) return null
    const p = JSON.parse(raw)
    if (typeof p.sha === 'string' && typeof p.contentSnapshot === 'string') return p as SyncState
  } catch { /* ignore */ }
  return null
}
```

Auto-pull-on-open effect — use the exact cancellation-guard pattern (empty deps, eslint-disable), guarding every setState with `if (!cancelled)`:
```
useEffect(() => {
  let cancelled = false
  ;(async () => {
    await Promise.resolve()
    if (cancelled) return
    const cfg = loadRepoConfig()
    if (!cfg || !token) return           // nothing configured or not signed in
    setPulling(true)
    try {
      const remote = await getFileContent(token, cfg.owner, cfg.repo, cfg.filePath, cfg.branch)
      if (cancelled) return
      const snap = loadSyncState()
      const local = contentRef.current
      if (!snap || local === snap.contentSnapshot) {
        // no local edits since last sync → take remote silently (D-07)
        applyRemoteContent(remote.content)
        setSyncState({ sha: remote.sha, contentSnapshot: remote.content })
      } else if (remote.content === local) {
        // identical content, just refresh sha/snapshot
        setSyncState({ sha: remote.sha, contentSnapshot: remote.content })
      } else {
        // local edited AND differs from remote → conflict (D-08)
        setConflict({ remoteContent: remote.content, remoteSha: remote.sha, source: 'pull' })
      }
    } catch (e) {
      if (!cancelled) setSyncError('Couldn’t sync with GitHub — working locally')
      // Note: 401 token-clearing is handled by useGitHubAuth on its own fetch; do not clear here.
    } finally {
      if (!cancelled) setPulling(false)
    }
  })()
  return () => { cancelled = true }
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [token])
```

```
if (!token || !repoConfig || !syncState) return
setCommitting(true); setSyncError(null)
try {
  const content = contentRef.current
  const result = await commitFile(token, repoConfig.owner, repoConfig.repo, repoConfig.filePath, repoConfig.branch, content, message, syncState.sha)
  setSyncState({ sha: result.sha, contentSnapshot: content })   // dirty clears: content === snapshot

} catch (e) {
  const msg = e instanceof Error ? e.message : ''
  if (msg.includes('commit_failed_409') || msg.includes('commit_failed_422')) {
    // sha mismatch — someone changed remote. Fetch fresh remote and re-raise the same conflict (D-13)
    try {
      const remote = await getFileContent(token, repoConfig.owner, repoConfig.repo, repoConfig.filePath, repoConfig.branch)
      setConflict({ remoteContent: remote.content, remoteSha: remote.sha, source: 'commit' })
    } catch {
      setSyncError('Commit failed — check your connection and try again')
    }
  } else {
    setSyncError('Commit failed — check your connection and try again')
  }
} finally {
  setCommitting(false)
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app state | repo config and sync snapshot are read back and drive commits |
| remote content → editor | pulled GitHub content replaces editor content |
| commit → remote overwrite | commit writes local content over the remote file |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Tampering | localStorage repo-config / sync-state | mitigate | `loadRepoConfig`/`loadSyncState` JSON.parse inside try/catch and validate every field type before use; malformed data yields `null`, never a thrown render |
| T-12-07 | Data loss (silent overwrite) | auto-pull + commit | mitigate | sha-based divergence check (D-07): remote is taken silently only when local equals the last snapshot; otherwise a blocking conflict is raised. Commit sends the stored sha; a 409/422 mismatch re-raises the conflict instead of clobbering |
| T-12-08 | Denial of Service (network hang) | auto-pull effect | mitigate | Effect is cancellation-guarded and never blocks the UI; pull failure sets a dismissible error and the editor keeps working from localStorage (D-10) |
</threat_model>
