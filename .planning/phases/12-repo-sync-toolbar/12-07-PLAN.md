1. Open the reported localhost URL and sign in with GitHub (Phase 11 flow). Confirm your avatar appears.
2. Open the **File** menu. Confirm it contains **Import MD** and **Download MD**, a divider, and **Connect repository…**. Confirm **Export PDF** is a SEPARATE button outside the menu. (SYNC-05/TOOL-01)
3. Click **Connect repository…**. In the dialog: type in the search box and confirm the repo list filters; pick a repo. 4. Confirm the header shows the `owner/repo · file.md` caption next to the md2cv title (desktop width). (SYNC-05)

6. Open **File → Commit to GitHub…**. Confirm the message is pre-filled "Update <filename>". Clear it and confirm the Commit button disables; type a message and click **Commit Changes**. Confirm a green "Committed to <branch>" toast appears and the amber dot clears. Verify on github.com that the commit landed. (SYNC-04)
7. Conflict test: on github.com, edit the same file directly and commit. Back in the app, make a DIFFERENT local edit, then reload the page. Confirm the blocking conflict modal "Your local edits differ from GitHub" appears with **Keep my local version** / **Use GitHub version**. Try **Use GitHub version** and confirm the editor shows the remote content. (SYNC-03)
8. Offline test (optional): disable network, reload. Confirm an amber "Couldn’t sync with GitHub — working locally" toast appears and the editor still loads your last content (no blocking). (SYNC-02/D-10)
