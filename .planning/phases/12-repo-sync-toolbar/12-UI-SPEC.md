## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, dirty dot offset from button edge |
| sm | 8px | Toolbar item gap (`gap-2`), menu item vertical padding (`py-2`) |
| md | 16px | Header horizontal padding (`px-4`), dialog section spacing |
| lg | 24px | Dialog panel padding (`p-6`) |
| xl | 32px | Toolbar button height (`h-8`) |
| 2xl | 48px | Header height (`h-12`) |
| 3xl | 64px | Modal max-width gutter on mobile |

Exceptions:
- Toolbar button height is 32px (`h-8`) — not 48px — intentional compact toolbar pattern
- Touch target for mobile: File menu button minimum interactive area 44px tall (achieved via `min-h-[44px]` on mobile or keeping `h-12` header touch zone); dirty dot does not need its own touch target
- Dirty dot: 6px diameter (not a spacing multiple — purely a visual indicator, not an interactive element)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| App title | 18px (text-lg) | 700 (font-bold) | 1.5 |
| Heading (dialog title, menu section label) | 16px (text-base) | 600 (font-semibold) | 1.25 |
| Body / buttons / menu items / repo-info text | 14px (text-sm) | 400 (normal) | 1.5 |
| Repo/branch header caption | 12px (text-xs) | 400 (normal) | 1.5 |

Exactly 4 sizes, 2 weights (400 and 600/700 collapsed to "semibold+" tier).

**Source:** Inferred from `Header.tsx` existing classes (`text-lg font-bold`, `text-sm`, `text-sm font-semibold`). 12px added for the compact repo caption (D-16) which must not compete with the app title.

## Color

All values are Tailwind v4 default color tokens unless noted.

| Role | Tailwind Token | Approx Hex | Usage |
|------|---------------|------------|-------|
| Dominant (60%) | gray-900 | #111827 | Header bar, modal panel background, dropdown background |
| Secondary (30%) | gray-700 | #374151 | Toolbar buttons, form inputs, secondary menu surfaces |
| Accent (10%) | blue-600 | #2563EB | Export PDF button, focus rings, active/selected row highlight in picker list |
| Destructive | red-950 / red-400 | #450a0a / #f87171 | Error toasts only |
| Warning | amber-400 | #fbbf24 | Dirty indicator dot, warning/offline toasts |

Accent reserved for:
1. Export PDF button (existing — do not add accent to File menu button)

**Source:** Derived from `Header.tsx` (`bg-gray-900`, `bg-gray-700`, `bg-blue-600`, `bg-red-950 border-red-800 text-red-400`). Green and amber added for new toast variants and dirty indicator.

## Component Inventory

New components introduced in this phase. Each must match existing button/dropdown patterns from `Header.tsx`.

### 1. File Menu Button (replaces Import MD + Download MD)

```
<button>
  class: h-8 px-3 rounded bg-gray-700 text-white text-sm border border-gray-600
         hover:bg-gray-600 transition-colors flex items-center gap-1 relative
  label: "File"
  aria-haspopup="menu"
  aria-expanded={menuOpen}
```

- Chevron icon (4×4, inline SVG) to the right of the label to signal dropdown
- When dirty: amber dot positioned `absolute top-1 right-1 w-1.5 h-1.5 rounded-full bg-amber-400`
- aria-label when dirty: "File — uncommitted changes"

### 2. File Dropdown Menu

Mirrors existing avatar dropdown pattern from `Header.tsx`.

```
<div role="menu">
  class: absolute left-0 top-full mt-1 min-w-[200px] z-50 bg-gray-900
         border border-gray-600 rounded shadow-lg
```

Menu sections (always present, state-adapted):

**File section (always visible):**
- Import MD (menuitem)
- Download MD (menuitem)

**Divider:** `<hr class="border-gray-700 my-1" />`

**GitHub sync section (state-dependent):**

| App state | Items shown |
|-----------|-------------|
| Signed out | `<span class="px-3 py-2 text-xs text-gray-500">Sign in to sync with GitHub</span>` (non-interactive) |
| Signed in, no repo | `Connect repository…` (menuitem, enabled) |
| Signed in, repo connected | `Connect repository…` (menuitem, enabled) + `Commit to GitHub…` (menuitem, enabled) |

Menu item base class: `w-full text-left px-3 py-2 text-sm text-gray-200 hover:bg-gray-700 cursor-pointer`
Menu item disabled class: `px-3 py-2 text-sm text-gray-500 cursor-default` (no hover)

### 3. Dialog/Modal Primitive

First modal in the app — establishes the reusable pattern for picker, conflict prompt, and commit dialog.

```
<!-- Backdrop -->
<div class="fixed inset-0 bg-black/60 z-50 flex items-center justify-center p-4"
     role="dialog" aria-modal="true" aria-labelledby="{dialog-title-id}">

  <!-- Panel -->
  <div class="bg-gray-900 border border-gray-600 rounded-lg shadow-xl w-full max-w-md p-6">
    <h2 id="{dialog-title-id}" class="text-base font-semibold text-white mb-4">
      {title}
    </h2>
    <!-- content slot -->
    <!-- footer -->
    <div class="flex justify-end gap-2 mt-6">
      <button class="h-8 px-3 rounded bg-gray-700 text-white text-sm border border-gray-600 hover:bg-gray-600 transition-colors">
        Cancel
      </button>
      <button class="h-8 px-3 rounded bg-blue-600 text-white text-sm border border-blue-500 hover:bg-blue-500 transition-colors">
        {primary action}
      </button>
    </div>
  </div>
</div>
```

Close on: Escape key, backdrop click, Cancel button. Focus traps inside panel while open.

### 4. Picker Dialog (SYNC-01)

Title: "Connect repository"
Max width: `max-w-lg`

**Step 1 — Repository selection:**
- Search input: `h-8 w-full px-3 rounded bg-gray-700 text-white text-sm border border-gray-600 focus:outline-none focus:ring-1 focus:ring-blue-400 placeholder:text-gray-500`
- Placeholder: "Search repositories…"
- Repo list: `max-h-56 overflow-y-auto mt-2 rounded border border-gray-700`
  - Row: `px-3 py-2 text-sm text-gray-200 hover:bg-gray-700 cursor-pointer flex items-center justify-between`
  - Selected row: `bg-blue-900/40 text-blue-200`
  - Last-pushed date: `text-xs text-gray-500`
- Loading state: spinner (`border-2 border-white border-t-transparent rounded-full w-4 h-4 animate-spin`) centered in list area, label "Loading repositories…" below spinner in `text-sm text-gray-400`

**Step 2 — Branch + file selection (same dialog, step advances after repo selection):**
- Branch dropdown: `h-8 px-2 rounded bg-gray-700 text-white text-sm border border-gray-600 focus:outline-none focus:ring-1 focus:ring-blue-400` — pre-selects default branch
- File list: same row pattern as repo list but shows `.md` files only; label "Select a file" above list in `text-xs text-gray-400 mb-1`

Primary CTA (step 1): "Next" — `bg-gray-700` (not accent — accent reserved for final confirm)
Primary CTA (step 2): "Open file" — `bg-blue-600`

### 5. Conflict Modal (SYNC-02, SYNC-03, SYNC-04 re-use)

Blocking — renders on top of picker if triggered on open. Cannot be dismissed without choosing.

Title: "Your local edits differ from GitHub"
Body: `class="text-sm text-gray-300 mb-6"` — "Choose which version to keep. This cannot be undone."

Two full-width action buttons (stacked vertically, `flex flex-col gap-2`):
- "Keep my local version" — `h-9 w-full rounded bg-gray-700 text-white text-sm border border-gray-600 hover:bg-gray-600 transition-colors`
- "Use GitHub version" — `h-9 w-full rounded bg-red-800 text-white text-sm border border-red-700 hover:bg-red-700 transition-colors` (destructive styling — overwrites local)

No Cancel. User must choose.

### 6. Commit Dialog (SYNC-04)

Title: "Commit to GitHub"

- Message input (`<input type="text">`): same input class as search input above
- Pre-filled value: `Update <filename>` (e.g., "Update resume.md")
- Label above input: `class="block text-xs text-gray-400 mb-1"` — "Commit message"
- Empty message: primary button disabled (`opacity-50 cursor-not-allowed`) — no commit without a message

Primary CTA: "Commit" — `bg-blue-600`
Cancel: `bg-gray-700`

### 7. Repo/Branch Caption in Header (SYNC-05, D-16)

Mounts between the `md2cv` title and the toolbar buttons, inside the header's left section.

```
<span class="text-xs text-gray-400 truncate max-w-[160px] hidden sm:inline-block ml-3">
  owner/repo · file.md
</span>
```

- Hidden on mobile (`hidden sm:inline-block`) — header is too cramped at < 640px
- Truncates with ellipsis when string exceeds 160px
- No tooltip required (Phase 13 tree will give full context)

### 8. Toast Variants (SYNC-04, D-14, D-10)

Base class (all toasts): `fixed top-12 right-4 z-50 flex items-center gap-2 px-3 py-2 rounded text-sm shadow-md max-w-[320px]`

| Variant | Additional classes |
|---------|-------------------|
| Error (existing) | `bg-red-950 border border-red-800 text-red-400` |
| Success | `bg-green-950 border border-green-800 text-green-400` |
| Warning / offline | `bg-amber-950 border border-amber-800 text-amber-400` |

All toasts include a dismiss `×` button: `ml-auto pl-2 hover:text-white cursor-pointer`

Auto-dismiss: success toasts after 4 seconds. Error and warning toasts require manual dismiss.

## Accessibility Contract

- All dialogs: `role="dialog" aria-modal="true" aria-labelledby="{id}"`
- Focus trapped inside open modal; restored to trigger element on close
- Escape closes menus and dialogs (except conflict modal — no Escape; user must choose)
- Dirty dot: conveys state via `aria-label` on the File button — not color alone
- Toasts: `role="alert" aria-live="polite"` (error/warning); success toasts `aria-live="polite"`
- Commit button disabled state: `aria-disabled="true"` when message empty
- Spinner: `aria-hidden="true"` on the visual spinner; button's accessible label updated to reflect in-progress state
