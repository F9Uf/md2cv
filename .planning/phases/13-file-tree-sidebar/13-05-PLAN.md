       ```typescript
       // Click a .md file in the tree. On mobile, close the drawer immediately (UI-SPEC §2).
       const handleTreeOpenFile = useCallback((path: string) => {
         if (!isDesktop) setSidebarOpen(false)
         if (repoSync.repoConfig && path === repoSync.repoConfig.filePath) return // already open
         if (repoSync.isDirty) {
           setPendingSwitchPath(path)          // opens DirtySwitchDialog
         } else {
           repoSync.openFile(path)
         }
       }, [isDesktop, repoSync])

       const currentFilename = repoSync.repoConfig
         ? (repoSync.repoConfig.filePath.split('/').pop() ?? 'file')
         : 'file'

       // DirtySwitchDialog actions
       const handleDirtyDiscard = useCallback(() => {
         const path = pendingSwitchPath
         setPendingSwitchPath(null)
         if (path) repoSync.openFile(path) // openFile replaces content with remote → discards local edits
       }, [pendingSwitchPath, repoSync])

       const handleDirtyCommit = useCallback(() => {
         awaitingCommitRef.current = true   // effect below performs the switch after a successful commit
         setPendingSwitchPath(prev => prev)  // keep pending path
         setCommitOpen(true)                 // open the Phase 12 commit dialog
       }, [])

       const handleDirtyCancel = useCallback(() => setPendingSwitchPath(null), [])
       ```

    5. Auto-switch after a commit initiated from the dirty prompt (add an effect). It fires the pending
       switch only once the commit finishes cleanly (committing back to false, dirty cleared, no error);

       ```typescript
       useEffect(() => {
         if (!awaitingCommitRef.current) return
         if (repoSync.committing) return            // still committing
         if (repoSync.syncError) {                  // commit failed → abort switch
           awaitingCommitRef.current = false
           setPendingSwitchPath(null)
           return
         }
         if (!repoSync.isDirty && pendingSwitchPath) { // commit succeeded → switch now
           const path = pendingSwitchPath
           awaitingCommitRef.current = false
           setPendingSwitchPath(null)
           repoSync.openFile(path)
         }
       }, [repoSync.committing, repoSync.isDirty, repoSync.syncError, pendingSwitchPath, repoSync])
       ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user tree interaction → editor content swap | a click can replace unsaved local edits |
| localStorage → app state | `md2cv-sidebar` string read at init |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-14 | Tampering | silent loss of uncommitted edits on file switch | mitigate | isDirty gate routes dirty switches through DirtySwitchDialog; openFile is only called on the clean path, on explicit Discard, or after a confirmed successful commit |
| T-13-15 | Tampering | corrupted `md2cv-sidebar` localStorage value | mitigate | read is wrapped in try/catch with a boolean default (`!== 'false'` → true); any non-'false' value is treated as open, never throws |
| T-13-16 | Repudiation | stale pending switch firing an unintended commit-triggered swap | mitigate | awaitingCommitRef cleared on CommitDialog onClose and on commit error; the switch effect requires both the ref flag and a non-dirty, error-free commit completion |
| T-13-17 | Information disclosure | sidebar/toggle rendered without a connected repo | mitigate | toggle gated on `repoConfig` in Header; FileSidebar `open` prop ANDed with `!!repoSync.repoConfig`; scrim gated on `repoSync.repoConfig` |
</threat_model>
