## Spacing Scale

Declared values (multiples of 4 only):

| Token | px  | Tailwind | Usage |
|-------|-----|----------|-------|
| xs    | 4px | gap-1 / p-1 | Icon gap within tree row, dirty dot offset |
| sm    | 8px | gap-2 / p-2 | Sidebar header internal gaps, refresh button padding |
| md    | 16px | gap-4 / px-4 | Header left padding (matches existing header px-4) |
| lg    | 24px | gap-6 / p-6 | Dialog body padding (matches existing Dialog p-6) |
| xl    | 32px | gap-8 | Major section breaks within dialog |
| 2xl   | 48px | h-12 | Header height, sidebar header row height |
| 3xl   | 64px | — | Not used in this phase |

**Tree row indent per depth level:** 12px (pl-3 per depth). Root items pl-2 (8px); depth 1 pl-5 (20px); depth 2 pl-8 (32px); depth 3 pl-11 (44px). Chosen to fit comfortably within the 240px fixed width.

**Tree row height:** 28px (h-7). VS Code compact row height. Gives ~10 rows visible at a glance in a typical viewport.

Exceptions:
- Toggle button in header: 32px height (h-8), 32px width (w-8) — icon-only, matches existing header button height; meets 44px touch target on mobile via Tailwind's `min-h-[44px]` override on the wrapping header zone.
- Mobile touch targets: minimum 44px tap height on tree rows when `(max-width: 767px)` — override tree row to h-11 (44px) on mobile.

## Interaction Contracts

### 1. Sidebar toggle (D-01, D-02, TREE-02)

- Toggle button: icon-only, h-8 w-8, placed immediately before the md2cv `<h1>` inside the header's left group.
- Icon: panel-left SVG (sidebar with left panel highlighted) — 16×16, viewBox="0 0 16 16". Draws a rectangle split into a left panel (filled) and right content area (outline).
- Visible only when `repoConfig !== null`. When no repo connected, button is absent (not just disabled).
- Click: toggles `sidebarOpen` boolean; persists to localStorage key `md2cv-sidebar` (serialize as `"true"` / `"false"`).
- localStorage init: lazy useState with try/catch, default `true` if key absent (auto-opens on first repo connect, per D-02).
- aria-label: "Toggle file tree sidebar" (static; no aria-expanded needed — the sidebar presence is visually obvious).
- aria-pressed not required (icon-only toggle in dense header; aria-label covers the state adequately).

### 2. Sidebar layout (D-03, D-04)

**Desktop (min-width: 768px):**
- `<FileSidebar>` renders as a fixed-width 240px flex column, inserted as `<aside>` before `<main>` inside the app shell flex row.
- When `sidebarOpen === false`: sidebar is hidden via `hidden` class (not unmounted — preserves tree fetch state).
- No resize handle; no transition on desktop open/close (instantaneous toggle matches VS Code behavior).

**Mobile (max-width: 767px):**
- Sidebar renders as an overlay drawer: `position: fixed; left: 0; top: 48px; height: calc(100vh - 48px); width: 240px; z-index: 40`.
- Opens/closes with CSS translate: `translate-x-0` (open) / `-translate-x-full` (closed).
- Transition: `transition-transform duration-200 ease-out`.
- Scrim: `<div class="fixed inset-0 bg-black/40 z-30">` rendered when drawer is open; click on scrim closes drawer.
- Picking a file in the tree closes the drawer immediately (before the file load starts).
- MobileTabs layout underneath is untouched.

### 3. Tree rendering (D-09, D-10, D-12)

**Fetch:**
- On repo connect: `useRepoTree` fetches `GET /repos/{owner}/{repo}/git/trees/{sha}?recursive=1` (unfiltered — all entries).
- On manual refresh (sidebar header refresh button): re-fetches same endpoint.
- No polling. No re-fetch on sidebar toggle.
- Very large repos: GitHub caps the recursive tree at ~100k items. If `response.truncated === true`, display inline notice below tree: "Repository too large — some folders may be missing." text-xs text-amber-400. Tree renders whatever was returned.

**Sorting:** At every level, folders first (alphabetical), then files (alphabetical). Case-insensitive sort.

**Expand/collapse:**
- Folders start collapsed, EXCEPT the path segments leading to the currently open file (`repoConfig.filePath`), which auto-expand on load.
- Expand state lives in component state (not localStorage). Resets when tree data is re-fetched.
- Clicking a folder row toggles expand/collapse. Click target is the full row width.
- Chevron: rotates 0° (collapsed) → 90° (expanded), `transition-transform duration-150`.

**Row anatomy (left to right):**
1. Indent spacer: `padding-left = depth * 12px + 8px` (base 8px)
2. Chevron (folders only): 12×12 SVG, text-gray-400, rotates on expand. Files have no chevron; file icon shifts left to align at the same x as folder icon.
3. Icon: 14×14 SVG — folder icon (closed/open variant) or file icon. text-gray-400.
4. Name: text-sm text-gray-200 truncate. For non-.md files: text-gray-500. For active row: text-white font-semibold.
5. Dirty dot (active .md row only, when isDirty): absolute or inline, w-1.5 h-1.5 rounded-full bg-amber-400, ml-auto mr-2.

**Clickability:**
- `.md` files: `cursor-pointer`, hover:bg-gray-700.
- Non-`.md` files: `cursor-not-allowed`, no hover highlight. `aria-disabled="true"` on the row.
- Folders: `cursor-pointer`, hover:bg-gray-700.

**Active row:** `bg-gray-700` background on the row whose path matches `repoConfig.filePath`.

### 4. Opening a file (TREE-03, TREE-04, D-05–D-08)

**Clean switch (isDirty === false):**
1. User clicks a `.md` file row.
2. Sidebar immediately shows a loading indicator on that row (spinner icon replaces file icon, or opacity-50 on the row).
3. `useRepoSync.openFile(path)` fetches `GET /repos/{owner}/{repo}/contents/{path}` with the current token.
4. On success: `applyRemoteContent(content)` is called, `repoConfig.filePath` updated, sync state re-anchored.
5. Loading indicator clears. Active row shifts to the newly opened file.
6. On mobile: drawer closes before step 2 (optimistic close).

**Dirty switch (isDirty === true):**
1. User clicks a different `.md` file row.
2. `DirtySwitchDialog` opens immediately. Tree row click does NOT trigger file load yet.
3. Dialog title: "Unsaved Changes"
4. Dialog body: "**{currentFilename}** has uncommitted changes. What would you like to do?"
5. Three actions (stacked, full-width buttons):
   - Primary (blue): "Commit changes…" — opens CommitDialog; on commit success, switches to the clicked file.
   - Destructive (red text, no background): "Discard my edits" — discards local edits, switches to clicked file immediately.
   - Neutral (gray): "Cancel" — closes dialog, stays on current file, no file switch.
6. Escape / backdrop click = Cancel.

**Failed load (D-07):**
- If fetch fails (network error, 401, 404): switch aborts. Current file and its edits are untouched.
- Error toast fires using existing sync-error toast pattern: text "Couldn't open file — check your connection and try again." (role="alert", top-right, bg-red-950 border-red-800 text-red-400).
- Active row in tree does NOT change to the failed-to-load file.

### 5. Loading state (sidebar body)

- While tree is fetching: render 5 skeleton rows at h-7 each.
- Skeleton row: `bg-gray-700 rounded animate-pulse h-3 w-3/4 ml-8` (mimics file name line).
- Skeleton rows have aria-hidden="true"; sidebar has `aria-busy="true"` while loading.

### 6. Empty state (no repo connected)

This state should never render because the sidebar/toggle are hidden when `repoConfig === null`. Included defensively.

- Heading: "No repository" — text-xs text-gray-500 text-center mt-8
- Body: "Connect a repository from File → Connect repository…" — text-xs text-gray-500 text-center px-4

### 7. Error state (tree fetch failed)

- Icon: ⚠ SVG or a simple text indicator
- Copy line 1: "Couldn't load file tree" — text-xs text-red-400
- Copy line 2: "Check your connection." — text-xs text-gray-500
- Retry button: "Refresh tree" — text-xs text-gray-400 underline cursor-pointer; triggers manual re-fetch.
- Rendered in the sidebar body below the header.

### 8. Sidebar header row

- Height: h-12 (48px) — matches main app header height.
- Background: bg-gray-900 (same as panel).
- Bottom border: border-b border-gray-700.
- Left content: "EXPLORER" label — text-xs uppercase tracking-widest text-gray-500 font-normal; NOT bold.
- Right content: refresh icon button — h-6 w-6, text-gray-400 hover:text-gray-200 cursor-pointer transition-colors; aria-label="Refresh file tree"; spins (animate-spin) while tree is loading.
