    Create src/components/FileSidebar.tsx as a controlled presentational container. Props:
    ```typescript
    import { useMediaQuery } from '../hooks/useMediaQuery'
    import FileTree from './FileTree'
    import type { TreeNode } from '../lib/fileTree'

    interface FileSidebarProps {
      open: boolean
      activePath: string | null    // repoConfig.filePath (or null)
      isDirty: boolean
      tree: TreeNode[]
      loading: boolean
      error: string | null
      truncated: boolean
      expandedPaths: Set<string>
      onToggleFolder: (path: string) => void
      onOpenFile: (path: string) => void
      onRefresh: () => void
    }
    ```
    Structure (per UI-SPEC):
    - `isDesktop = useMediaQuery('(min-width: 768px)')`.
    - Root element is `<aside aria-label="File tree" aria-busy={loading}>`.
      - Desktop: className `w-60 shrink-0 flex flex-col bg-gray-900 border-r border-gray-700` when `open`,
        else `hidden` (CSS-hidden, NOT unmounted, so tree fetch state survives toggling).
      - Mobile: className is a fixed overlay drawer:
        `fixed left-0 top-12 z-40 w-60 h-[calc(100vh-3rem)] flex flex-col bg-gray-900 border-r border-gray-700 transition-transform duration-200 ease-out` plus
        `${open ? 'translate-x-0' : '-translate-x-full'}`. (The scrim + drawer-close-on-file-pick are
        wired by App in Plan 05; this component only renders the panel and its translate state.)
    - Header row: `<div className="h-12 shrink-0 flex items-center justify-between px-3 bg-gray-900 border-b border-gray-700">`
      - Left: `<span className="text-xs uppercase tracking-widest text-gray-500">EXPLORER</span>` (weight normal, NOT bold).
      - Right: refresh `<button aria-label="Refresh file tree" onClick={onRefresh}` classes
        `h-6 w-6 flex items-center justify-center rounded text-gray-400 hover:text-gray-200 hover:bg-gray-700 transition-colors`.
        Refresh SVG (16×16, currentColor, aria-hidden) gets `className={loading ? 'animate-spin' : ''}` so it spins while loading.
        Keep the aria-label as "Refresh file tree" even while spinning (do NOT change to "Loading").
    - Body `<div className="flex-1 overflow-auto py-1">` renders exactly one of:
      1. `loading` → 5 skeleton rows, each `<div aria-hidden="true" className="h-7 flex items-center"><div className="bg-gray-700 rounded animate-pulse h-3 w-3/4 ml-8" /></div>`.
      2. `error` → error state: line1 `<p className="text-xs text-red-400 ...">Couldn't load file tree</p>`,
         line2 `<p className="text-xs text-gray-500 ...">Check your connection.</p>`, and a retry
         `<button onClick={onRefresh} className="text-xs text-gray-400 underline cursor-pointer">Refresh tree</button>`.
      3. `tree.length === 0` (no error, not loading) → empty note `<p className="text-xs text-gray-500 text-center mt-8 px-4">No repository</p>` /
         `<p className="text-xs text-gray-500 text-center px-4">Connect a repository from File → Connect repository…</p>` (defensive; normally hidden by App gating).
      4. otherwise → `<FileTree nodes={tree} activePath={activePath} isDirty={isDirty} expandedPaths={expandedPaths} onToggleFolder={onToggleFolder} onOpenFile={onOpenFile} />`,
         and when `truncated`, render below it `<p className="text-xs text-amber-400 px-3 py-1">Repository too large — some folders may be missing.</p>`.
    - Copy strings must match the Copywriting Contract exactly (EXPLORER, "Couldn't load file tree",
      "Check your connection.", "Refresh tree", "No repository", the connect copy, and the truncated notice).
