<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    The file-tree sidebar: a toggleable VS Code-style directory tree (Header toggle + FileSidebar +
    FileTree + useRepoTree), click-to-open markdown files via useRepoSync.openFile, a dirty-switch
    Commit/Discard/Cancel prompt, and a mobile overlay drawer.
  </what-built>
  <how-to-verify>
    Sign in with GitHub and connect a repository that contains nested folders and a mix of .md and
    non-.md files. Then check each:

    TREE-01 (tree renders):
    1. The sidebar appears on the left showing the repo's full directory tree.
    2. Folders sort before files, alphabetical at every level (VS Code order).
    3. Non-markdown files (e.g. .png, .json) appear grayed out and are NOT clickable.
    4. Clicking a folder expands/collapses it; the chevron rotates. The folder path to the currently
       open file is auto-expanded on load.
    5. Click the refresh icon in the EXPLORER header — it spins and the tree re-fetches.

    TREE-02 (toggle + persistence):
    6. Click the panel icon at the far left of the header — the sidebar hides; click again — it shows.
    7. Hide the sidebar, then RELOAD the page — it stays hidden. Show it, reload — it stays shown.
    8. Disconnect / sign out: the toggle button disappears when no repo is connected.

    TREE-03 (click opens file):
    9. Click a .md file in the tree — the editor and preview immediately switch to that file's content.
    10. The clicked row becomes the highlighted active row.

    TREE-04 (single file replaces + dirty guard):
    11. Edit the open file (make it dirty — amber dot shows on the active row), then click a different
        .md file. The "Unsaved Changes" dialog appears with Commit changes… / Discard my edits / Cancel.
    12. Cancel → stays on the current file, edits intact.
    13. Discard my edits → switches to the clicked file (your unsaved edits are gone, as intended).
    14. Make edits again, click another file, choose Commit changes… → commit dialog opens; after a
        successful commit the app switches to the clicked file.
    15. Only ever one file is open — opening a new one replaces the previous.

    Mobile (resize the browser to a phone width, < 768px, or use device emulation):
    16. The sidebar is an overlay drawer sliding in from the left with a dark scrim behind it.
    17. Picking a file closes the drawer; tapping the scrim closes it.

    Failure safety:
    18. (Optional) Go offline and click a file — an error toast "Couldn't open file — check your
        connection and try again." appears and the current file/edits are untouched.
  </how-to-verify>
  <resume-signal>Type "approved" if all criteria pass, or describe any issues found.</resume-signal>
</task>
